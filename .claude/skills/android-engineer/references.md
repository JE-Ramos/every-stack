## RTFM — Essential Reading

### Official Documentation
- [Android Developer Guides](https://developer.android.com/develop) — Architecture components, Compose, lifecycle, and best practices. Start with the Guide to App Architecture.
- [Jetpack Compose documentation](https://developer.android.com/develop/ui/compose) — Layouts, state, side effects, animation, navigation, testing. Read the mental model and thinking-in-Compose guides first.
- [Kotlin documentation](https://kotlinlang.org/docs/home.html) — Coroutines, Flow, sealed classes, and language features. The coroutines guide is essential.
- [Material 3 for Compose](https://developer.android.com/develop/ui/compose/designsystems/material3) — Theming, components, dynamic color, and adaptive design.

### Books
- **Kotlin in Action (2nd Edition)** by Roman Elizarov et al. — Deep understanding of Kotlin including coroutines
- **Kotlin Coroutines: Deep Dive** by Marcin Moskala — The definitive guide to structured concurrency, Flow, and coroutine internals
- **Atomic Kotlin** by Bruce Eckel & Svetlana Isakova — Excellent for solidifying Kotlin fundamentals

### Articles, Blogs & Channels
- [Now in Android](https://github.com/android/nowinandroid) — Google's reference Compose application demonstrating architecture, modularization, build system conventions, and testing strategies. Study this before building anything new.
- [Philipp Lackner](https://www.youtube.com/@PhilippLackner) — Practical, up-to-date Compose and Android architecture tutorials
- [Android Developers Blog](https://android-developers.googleblog.com) — Official platform updates, Compose releases, and tooling announcements
- [Styling Android](https://blog.stylingandroid.com) — Deep dives on Compose animation and custom drawing
- [ProAndroidDev on Medium](https://proandroiddev.com) — Community articles on architecture, Compose patterns, and Kotlin techniques
- [Jake Wharton's blog](https://jakewharton.com) — Insights from one of Android's most influential engineers on tooling, Kotlin, and Compose internals
- [Compose Samples](https://github.com/android/compose-samples) — Official sample apps: Jetsnack, Rally, Crane, Jetchat. Reference implementations of common patterns.

### Tools & References
- [detekt documentation](https://detekt.dev) — Configuration, custom rules, and IDE integration
- [compose-rules](https://github.com/mrmans0n/compose-rules) — Compose-specific static analysis rules for detekt and lint
- [Gradle Build Scans](https://scans.gradle.com) — Diagnose build performance issues
- [LeakCanary](https://square.github.io/leakcanary/) — Memory leak detection during development
- [Turbine](https://github.com/cashapp/turbine) — Testing library for Kotlin Flow
