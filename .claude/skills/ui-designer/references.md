## RTFM — Essential Reading

### Books
- [Refactoring UI](https://www.refactoringui.com/) — Adam Wathan & Steve Schoger. The most practical visual design book for product interfaces. Specific, actionable tactics for hierarchy, spacing, typography, color, and depth
- [Design Systems](https://www.smashingmagazine.com/design-systems-book/) — Alla Kholmatova. How to build and maintain design systems that connect people — governance, principles, patterns, and team dynamics
- [Atomic Design](https://atomicdesign.bradfrost.com/) — Brad Frost. Free online. The methodology for building component-based design systems from atoms to pages
- [The Elements of Typographic Style](https://typographica.org/typography-books/the-elements-of-typographic-style-4th-edition/) — Robert Bringhurst. The authoritative text on typography — essential for anyone designing type systems
- [Grid Systems in Graphic Design](https://www.niggli.ch/en/grid-systems-in-graphic-design.html) — Josef Muller-Brockmann. Classic grid and layout design principles that remain foundational for digital design
- [Interaction of Color](https://yalebooks.yale.edu/book/9780300179354/interaction-of-color/) — Josef Albers. The classic text on how colors behave in context — essential for palette design

### Documentation & Guides
- [Material Design 3](https://m3.material.io/) — Google's design system documentation. Comprehensive component specs, token architecture, and design guidance
- [Apple Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines) — Platform-specific design patterns, including typography, color, and layout systems
- [Figma Best Practices](https://www.figma.com/best-practices/) — Official guide to component architecture, auto-layout, and variables
- [Style Dictionary Documentation](https://amzn.github.io/style-dictionary/) — Design token transformation framework — how tokens flow from design to code
- [Tokens Studio Documentation](https://docs.tokens.studio/) — Figma plugin for design token management and multi-theme support

### Articles & Resources
- [Brad Frost's Blog](https://bradfrost.com/blog/) — Design systems thinking from the author of Atomic Design
- [Nathan Curtis on Medium](https://medium.com/@nathanacurtis) — Deep articles on design system component design, documentation, and governance
- [SmashingMagazine Design Systems](https://www.smashingmagazine.com/category/design-systems/) — Practical articles on building and scaling design systems
- [Type Scale](https://typescale.com/) — Interactive tool for generating modular type scales
- [Contrast Grid](https://contrast-grid.eightshapes.com/) — Tool for checking contrast ratios across your entire color palette at once
- [Open Props](https://open-props.style/) — CSS custom properties for design tokens — useful reference for token naming and scale design

### OSS Projects & Design Systems
- [Radix UI](https://www.radix-ui.com/) — Unstyled, accessible component primitives for building design systems
- [shadcn/ui](https://ui.shadcn.com/) — Beautifully designed components built with Radix and Tailwind — excellent component pattern reference
- [Storybook](https://storybook.js.org/) — Component development, documentation, and testing environment
- [Figma Community Design Systems](https://www.figma.com/community/tag/design%20system/files) — Open design system files for reference and learning
